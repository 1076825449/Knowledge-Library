# CONTENT_REVIEW_PLAYBOOK.md

> 更新：2026-04-22
> 目的：给内容复审建立可执行节奏，避免“导入完就算结束”

---

## 1. 复审对象优先级

优先复审以下内容：

1. `high_frequency_flag = 1` 的问题
2. `newbie_flag = 1` 的入口问题
3. `certain_condition` / `certain_dispute` / `certain_practice` 的边界问题
4. 地方口径问题
5. 最近一轮批量扩写导入的问题

## 2. 单条卡片复审清单

- 一句话结论有没有直接回答问题
- 适用条件是否具体
- 例外与边界是否足够清楚
- 实务步骤是否能执行
- 风险提示是否写到业务后果
- 政策依据是否和结论对应
- 标签是否能支撑检索
- 关联问题是否能把用户带到下一步

## 3. 抽检比例

- 新导入批次：至少抽检 20%
- 高频问题：优先 100% 过一遍
- 一般问题：可按模块分批复审

## 4. 复审节奏建议

- 每周：抽检最近新增和高频入口
- 每月：按专题簇复审一个主线
- 每季度：回看争议问题、地方口径和风险问题

## 5. 复审输出

复审后至少要留下：

- 复审日期
- 复审人
- 是否通过
- 是否需要修订
- 是否需要补政策、补边界、补关联或降级对外表述
